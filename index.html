	}

    shorthand: four properties

    #boxModel {
	padding: 100px 75px 50px 25px;
	}

    single property:

    #boxModel {
	padding-top: 100px;
	}

    four sides equally padded:

    #boxModel {
	padding: 100px;
	}

    two sides with equal padding:

    #boxModel {
	padding: 100px 15px;
	}
  6. something to note here, in the shorthand there are NO commas.
  7. shorthand goes in clockwise order: TOP, RIGHT, BOTTOM, LEFT

inline elements vs. block elements

what's the difference?

block-level elements
will always start on a new line, stacking one on top of the next

examples:

  • <h1 thru h6>
  • <p>
  • <ol>
  • <ul>
  • <li>
  • <article>
  • <div>
  • <footer>
  • <nav>
  • <section>
inline elements
will always appear to continue on the same line as their neighboring elements

examples:

  • <a>
  • <br>
  • <code>
  • <em>
  • <img>
  • <span>
  • <strong>
  • the behavior is different for inline and block elements:
type width & height padding & margins
block-level yes can apply to all four sides
inline depends will only affect the sides
  • what does this mean though?
  • hover on each of the images below, then take a look at the code samples

 margin-left: 5%;

 margin-top: 5%;

  • block level elements move regardless of what is around them.
  • inline elements can push their neighbors to the left or the right.
  • a single top or bottom property will affect all inline elements equally
  • but something strange happens when you push along the X and Y axis
